# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

Thin overlay image on top of `docker.io/searxng/searxng:latest` that adds Slovenian (`sl`, `sl-SI`) to the search language dropdown. Published to GHCR via CI, consumed by Coolify as a drop-in replacement for upstream SearXNG.

## Commands

```sh
docker compose up -d --build   # run stack locally (searxng + redis)
docker compose build --pull    # rebuild pulling latest upstream base
```

CI (`.github/workflows/build-push.yml`) builds multi-arch images on push to `master`, tags `v*`, weekly Monday 04:00 UTC, and manual dispatch. Publishes to `ghcr.io/<owner>/searxng-sl`.

## Architecture

### Why this repo exists

Upstream `searx/sxng_locales.py` is auto-generated by `./manage data.traits` (run inside a checkout of the upstream `searxng/searxng` repo), which runs `searxng_extra/update/update_engine_traits.py`. The `filter_locales()` function applies thresholds: `min_eng_per_lang=22`, `min_eng_per_region=18`. Slovenian has live support from only 19 engines (`sl`) and 14 (`sl-SI`), so it gets filtered out. This repo ships a patched `sxng_locales.py` with `sl` + `sl-SI` appended.

### Regenerating `sxng_locales.py`

Add a one-line patch in searxng repository at `searxng_extra/update/update_engine_traits.py:139` adding `sxng_tag_list.update(['sl', 'sl-SI'])` to force inclusion. To regen (in searxng repository):

```sh
./manage pyenv.install   # first time only
./manage data.traits     # hits live engines, ~10 min
```

Then copy `searx/sxng_locales.py` to `searxng-sl` repo.

`engine_traits.json` does not need copying — raw live data, unchanged by the patch.

### `.pyc` cache gotcha (critical)

Upstream builder runs `python -m compileall --invalidation-mode=unchecked-hash ./searx/` (PEP 552). That mode makes Python always load the cached `.pyc` regardless of `.py` mtime or hash. Simply `COPY`-ing a new `sxng_locales.py` has **zero effect** — the stale `.pyc` from the base image wins.

The Dockerfile therefore deletes `sxng_locales*.pyc` and recompiles with the same `unchecked-hash` mode after the copy. Any future file overrides (e.g. `engine_traits.json`) must follow the same pattern.

### Container layout

- SearXNG source: `/usr/local/searxng/searx/` (not pip-installed; copied as directory)
- Venv: `/usr/local/searxng/.venv/` — use `/usr/local/searxng/.venv/bin/python` for compile steps
- Runtime user: `searxng` (uid 977). `USER root` needed for `compileall`; switch back after.
- Entrypoint: `/usr/local/searxng/entrypoint.sh`, runs `granian searx.webapp:app`

### Deploy (Coolify)

Deploy Coolify's built-in SearXNG one-click app, then edit its compose and swap `image: searxng/searxng:latest` → `image: ghcr.io/<owner>/searxng-sl:latest`. Keep volumes, env, redis sidecar. Config (`settings.yml`) is pasted manually into the volume via Coolify's file editor — not baked into this image.

GHCR is private. Coolify has no UI for registry auth — SSH the host and `docker login ghcr.io -u <user> --password-stdin` as the user Coolify runs as. Creds land in `~/.docker/config.json`, auto-detected on next pull.

Fine-grained GitHub PATs do **not** support GHCR. Use a classic PAT with `read:packages` scope only.
